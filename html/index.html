Model-driven agents on Spring Boot

Minimal setup, maximum autonomy

Add the SDK dependency, configure your model provider in application.yml, and expose an endpoint. The agent handles the rest: planning, tool selection, iteration, and response synthesis.

No DAGs to define. No routing logic to maintain. The foundation model decides which tools to call and when to stop. You focus on the goal, not the graph.

  • ✓ Auto-discovers all registered tool beans
  • ✓ Session management built in
  • ✓ Works with any Spring AI model starter
  • ✓ Production-ready security defaults
// application.yml: strands.agent.model-provider: bedrock // Just add a Spring AI model starter and configure credentials. @RestController public class AgentController { private final StrandsAgent agent; @GetMapping("/ask") public String ask(@RequestParam String q) { return agent.execute(q, StrandsExecutionContext.standalone("session-1") ).content(); } }

Fits into the Spring AI ecosystem

spring-ai-agentcore

Amazon Bedrock AgentCore

Natively imports the spring-ai-agentcore BOM for serverless runtimes, code interpreters, and browser automation tools managed by AWS.

spring-ai-a2a

Agent-to-Agent Protocol

Expose your agent as an A2A-compatible service. Other agents discover and invoke it via the standard Agent-to-Agent protocol.

spring-ai-mcp

Model Context Protocol

Connect to any MCP server. The SDK auto-discovers MCP tool providers and registers them in the execution loop alongside native tools.

Get started in minutes

1. Add the Maven dependency

<!-- pom.xml --> <dependency> <groupId>io.github.vaquarkhan</groupId> <artifactId>ai-agent-java-sdk-core</artifactId> <version>0.1.0-SNAPSHOT</version> </dependency>

2. Configure application.yml

# application.yml strands: agent: model-provider: bedrock system-prompt: "You are a helpful assistant." max-iterations: 10 spring: ai: bedrock: aws: region: us-east-1
